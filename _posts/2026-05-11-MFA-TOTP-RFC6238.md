## Hook

Google Authenticator가 보여주는 "847 291"이라는 6자리 숫자. 30초 후에는 완전히 다른 숫자가 됩니다. 이 숫자는 인터넷에 연결되지 않은, 완전히 오프라인인 스마트폰에서 생성됩니다. 어떻게 서버와 스마트폰이 같은 숫자를 동시에 만들어낼 수 있을까요? 비밀은 "같은 비밀키 + 같은 시간"입니다.

## TL;DR

- MFA는 인증의 3요소(Know, Have, Are) 중 2가지 이상을 조합합니다
- **TOTP(RFC 6238)** = 비밀키 + 현재 시간 → 6자리 코드 (30초마다 변경)
- 서버와 스마트폰이 **같은 비밀키**를 공유하므로 통신 없이도 같은 코드를 생성합니다
- TOTP는 **무료, 오프라인, Google Authenticator 호환** — 에어갭 환경에서도 작동합니다

## Solution: TOTP 작동 원리

**TOTP**는 "현재 시간"과 "비밀 키"를 조합하여 6자리 숫자를 생성하는 알고리즘입니다. RFC 6238 표준이며, Google Authenticator, Microsoft Authenticator가 사용하는 방식입니다.

```
  ┌──────────┐                    ┌──────────┐
  │  서버     │                    │  스마트폰  │
  │          │                    │  (Auth)   │
  │ 비밀키 K  │  ← 같은 비밀키 K →  │ 비밀키 K  │
  │ 시간 T   │  ← 같은 시간 T →   │ 시간 T   │
  └────┬─────┘                    └────┬─────┘
       │                               │
       │ TOTP(K, T) = 6자리 코드       │ TOTP(K, T) = 6자리 코드
       ▼                               ▼
  ┌──────────┐                    ┌──────────┐
  │ 847 291  │  ← 같은 6자리 →    │ 847 291  │
  └──────────┘                    └──────────┘

  핵심: 같은 비밀키 + 같은 시간 = 같은 코드
  30초마다 코드가 변경됨
```

**왜 안전한가?**

- 비밀번호가 유출되어도 OTP 코드가 없으면 로그인 불가
- OTP 코드는 30초마다 변경 → 가로채도 금방 만료
- 비밀키는 스마트폰에만 저장 → 서버가 해킹당해도 안전
- 오프라인 작동 (인터넷/통신 불필요)

### TOTP 알고리즘 (RFC 6238)

```python
import hmac, hashlib, struct, time

def totp(secret: bytes, interval: int = 30, digits: int = 6) -> str:
    """TOTP 코드 생성"""
    # 1. 현재 시간을 30초 단위로 나눔
    counter = int(time.time()) // interval

    # 2. 카운터를 바이트로 변환
    msg = struct.pack(">Q", counter)

    # 3. HMAC-SHA1 해시 계산
    h = hmac.new(secret, msg, hashlib.sha1).digest()

    # 4. 동적 자르기 → 6자리 숫자
    offset = h[-1] & 0x0F
    code = struct.unpack(">I", h[offset:offset+4])[0] & 0x7FFFFFFF
    return str(code % (10 ** digits)).zfill(digits)

# 사용 예:
secret = b"JBSWY3DPEHPK3PXP"  # Base32 디코딩된 비밀키
code = totp(secret)
print(f"현재 OTP 코드: {code}")  # 예: 847291
```

### QR 코드 등록

사용자가 처음 MFA를 등록할 때, QR 코드를 스캔하여 비밀키를 스마트폰에 저장합니다.

```
otpauth://totp/AD%20Manager:user@example.local
  ?secret=JBSWY3DPEHPK3PXP
  &issuer=AD%20Manager
  &digits=6
  &period=30
```

### 백업 코드

스마트폰을 분실했을 때를 대비한 일회용 코드입니다. 등록 시 1회만 표시되며, 각 코드는 1회만 사용 가능합니다.

<details markdown="1">
<summary>MFA 방식 비교표</summary>

| 방식 | 보안성 | 편의성 | 비용 | 오프라인 |
|------|--------|--------|------|---------|
| **TOTP** (Google Auth) | 높음 | 높음 | 무료 | O |
| SMS OTP | 중간 (SIM 스와핑 위험) | 높음 | 유료 (통신비) | X |
| 이메일 OTP | 중간 | 중간 | 무료 | X |
| Push 알림 | 높음 | 높음 | 유료 (앱 백엔드) | X |
| WebAuthn (FIDO2) | 최고 | 중간 | 하드웨어 필요 | O |
| 인증서 (Smart Card) | 최고 | 낮음 | 하드웨어 필요 | O |

**TOTP 선택 이유**: 무료, 오프라인 작동, Google Authenticator 호환, Python `pyotp` 라이브러리로 간단 구현. 에어갭 환경(인터넷 없는 AD)에서도 작동합니다.

</details>

## Takeaway

1. **TOTP는 "시간 동기화"에 의존한다** — 서버와 스마트폰의 시계가 30초 이상 어긋나면 인증이 실패합니다. NTP 동기화는 선택이 아닌 필수입니다
2. **비밀키 분실 = 영구 잠금** — QR 코드를 스캔한 후 비밀키를 다시 볼 수 없습니다. 백업 코드 외에 복구 수단이 없으므로, 백업 코드 생성은 필수입니다
3. **에어갭 환경에서 TOTP가 유일한 현실적 MFA** — SMS, Push, 이메일은 모두 인터넷이 필요합니다. 인터넷이 차단된 환경에서는 TOTP가 유일하게 작동하는 MFA 방식입니다
